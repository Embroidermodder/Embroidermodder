Embroidermodder is a free machine embroidery software program. The newest version, Embroidermodder 2 can\+:


\begin{DoxyItemize}
\item edit and create embroidery designs
\item estimate the amount of thread and machine time needed to stitch a design
\item convert embroidery files to a variety of formats
\item upscale or downscale designs
\item run on Windows, Mac and Linux
\end{DoxyItemize}

For more information, see \href{http://www.libembroidery.org}{\texttt{ our website}}.\hypertarget{md_README_autotoc_md0}{}\doxysection{Dependencies}\label{md_README_autotoc_md0}
To build Embroidermodder 2 from source you will need at least\+:


\begin{DoxyItemize}
\item The \href{https://github.com/Embroidermodder/Embroidermodder}{\texttt{ Embroidermodder 2}} source code.
\item \href{http://www.qt-project.org}{\texttt{ Qt}} (version \texorpdfstring{$>$}{>}= 6.\+0).
\item \href{http://www.opengl.org}{\texttt{ Open\+GL}} (version \texorpdfstring{$>$}{>}= 2.\+0).
\end{DoxyItemize}

Optionally, you may want to add\+:


\begin{DoxyItemize}
\item \href{http://www.kde.org}{\texttt{ KDE}} for thumbnailer.
\item \href{http://www.valgrind.org}{\texttt{ Valgrind}} for debugging.
\item \href{http://cmake.org}{\texttt{ CMake}} (version \texorpdfstring{$>$}{>}= 2.\+0) for build (see Building Without CMake below if this is an issue).
\end{DoxyItemize}

Ubuntu repository packages\+: The Qt, KDE and Valgrind build dependencies can be installed easily by opening a terminal and issuing this command\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{sudo apt-\/get install git build-\/essential qt6-\/dev-\/tools libqt6-\/opengl-\/dev \(\backslash\) }
\DoxyCodeLine{    valgrind cmake}

\end{DoxyCode}


Fedora repository packages\+: The Qt, KDE and Valgrind build dependencies can be installed easily by opening a terminal and issuing this command\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{sudo yum install git gdb gcc-\/c++ qt-\/devel kdelibs-\/devel valgrind}

\end{DoxyCode}

\hypertarget{md_README_autotoc_md5}{}\doxysubsection{Testing One-\/liner (\+Bash-\/only)}\label{md_README_autotoc_md5}
Sometimes an issue you\textquotesingle{}re having is fixed in the current version\+: it is recommended that you use a fresh build to ensure that the problem is still present before reporting this. This one-\/liner helps with this\+: place it in your {\ttfamily .bashrc} for future use.


\begin{DoxyCode}{0}
\DoxyCodeLine{alias run\_current\_em2="{}rm -\/f build.sh \&\& wget https://raw.githubusercontent.com/Embroidermodder/Embroidermodder/refs/heads/main/build.sh \&\& bash build.sh -\/-\/current"{}}

\end{DoxyCode}


Then, if you have issues, you can capture the terminal output like this.


\begin{DoxyCode}{0}
\DoxyCodeLine{run\_current\_em2 \&> build.log}

\end{DoxyCode}
