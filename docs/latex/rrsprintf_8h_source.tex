\DoxyCodeLine{4 \textcolor{comment}{/*}}
\DoxyCodeLine{5 \textcolor{comment}{Single file sprintf replacement.}}
\DoxyCodeLine{6 \textcolor{comment}{}}
\DoxyCodeLine{7 \textcolor{comment}{Originally written by Jeff Roberts at RAD Game Tools -\/ 2015/10/20. }}
\DoxyCodeLine{8 \textcolor{comment}{Hereby placed in public domain.}}
\DoxyCodeLine{9 \textcolor{comment}{}}
\DoxyCodeLine{10 \textcolor{comment}{This is a full sprintf replacement that supports everything that}}
\DoxyCodeLine{11 \textcolor{comment}{the C runtime sprintfs support, including float/double, 64-\/bit integers,}}
\DoxyCodeLine{12 \textcolor{comment}{hex floats, field parameters (\%*.*d stuff), length reads backs, etc.}}
\DoxyCodeLine{13 \textcolor{comment}{}}
\DoxyCodeLine{14 \textcolor{comment}{Why would you need this if sprintf already exists?  Well, first off,}}
\DoxyCodeLine{15 \textcolor{comment}{it's *much* faster (see below). It's also much smaller than the CRT}}
\DoxyCodeLine{16 \textcolor{comment}{versions code-\/space-\/wise. We've also added some simple improvements }}
\DoxyCodeLine{17 \textcolor{comment}{that are super handy (commas in thousands, callbacks at buffer full,}}
\DoxyCodeLine{18 \textcolor{comment}{for example). Finally, the format strings for MSVC and GCC differ }}
\DoxyCodeLine{19 \textcolor{comment}{for 64-\/bit integers (among other small things), so this lets you use }}
\DoxyCodeLine{20 \textcolor{comment}{the same format strings in cross platform code.}}
\DoxyCodeLine{21 \textcolor{comment}{}}
\DoxyCodeLine{22 \textcolor{comment}{It uses the standard single file trick of being both the header file}}
\DoxyCodeLine{23 \textcolor{comment}{and the source itself. If you just include it normally, you just get }}
\DoxyCodeLine{24 \textcolor{comment}{the header file function definitions. To get the code, you include}}
\DoxyCodeLine{25 \textcolor{comment}{it from a C or C++ file and define RR\_SPRINTF\_IMPLEMENTATION first.}}
\DoxyCodeLine{26 \textcolor{comment}{}}
\DoxyCodeLine{27 \textcolor{comment}{It only uses va\_args macros from the C runtime to do it's work. It}}
\DoxyCodeLine{28 \textcolor{comment}{does cast doubles to S64s and shifts and divides U64s, which does }}
\DoxyCodeLine{29 \textcolor{comment}{drag in CRT code on most platforms.}}
\DoxyCodeLine{30 \textcolor{comment}{}}
\DoxyCodeLine{31 \textcolor{comment}{It compiles to roughly 8K with float support, and 4K without.}}
\DoxyCodeLine{32 \textcolor{comment}{As a comparison, when using MSVC static libs, calling sprintf drags}}
\DoxyCodeLine{33 \textcolor{comment}{in 16K.}}
\DoxyCodeLine{34 \textcolor{comment}{}}

\DoxyCodeLine{58 \textcolor{comment}{FLOATS/DOUBLES:}}
\DoxyCodeLine{59 \textcolor{comment}{===============}}
\DoxyCodeLine{60 \textcolor{comment}{This code uses a internal float-\/>ascii conversion method that uses}}
\DoxyCodeLine{61 \textcolor{comment}{doubles with error correction (double-\/doubles, for \string~105 bits of}}
\DoxyCodeLine{62 \textcolor{comment}{precision).  This conversion is round-\/trip perfect -\/ that is, an atof}}
\DoxyCodeLine{63 \textcolor{comment}{of the values output here will give you the bit-\/exact double back.}}
\DoxyCodeLine{64 \textcolor{comment}{}}
\DoxyCodeLine{65 \textcolor{comment}{One difference is that our insignificant digits will be different than }}
\DoxyCodeLine{66 \textcolor{comment}{with MSVC or GCC (but they don't match each other either).  We also }}
\DoxyCodeLine{67 \textcolor{comment}{don't attempt to find the minimum length matching float (pre-\/MSVC15 }}
\DoxyCodeLine{68 \textcolor{comment}{doesn't either).}}
\DoxyCodeLine{69 \textcolor{comment}{}}
\DoxyCodeLine{70 \textcolor{comment}{If you don't need float or doubles at all, define RR\_SPRINTF\_NOFLOAT}}
\DoxyCodeLine{71 \textcolor{comment}{and you'll save 4K of code space.}}
\DoxyCodeLine{72 \textcolor{comment}{}}
\DoxyCodeLine{73 \textcolor{comment}{64-\/BIT INTS:}}
\DoxyCodeLine{74 \textcolor{comment}{============}}
\DoxyCodeLine{75 \textcolor{comment}{This library also supports 64-\/bit integers and you can use MSVC style or}}
\DoxyCodeLine{76 \textcolor{comment}{GCC style indicators (\%I64d or \%lld).  It supports the C99 specifiers}}
\DoxyCodeLine{77 \textcolor{comment}{for size\_t and ptr\_diff\_t (\%jd \%zd) as well.}}
\DoxyCodeLine{78 \textcolor{comment}{}}
\DoxyCodeLine{79 \textcolor{comment}{EXTRAS:}}
\DoxyCodeLine{80 \textcolor{comment}{=======}}
\DoxyCodeLine{81 \textcolor{comment}{Like some GCCs, for integers and floats, you can use a ' (single quote)}}
\DoxyCodeLine{82 \textcolor{comment}{specifier and commas will be inserted on the thousands: "{}\%'d"{} on 12345 }}
\DoxyCodeLine{83 \textcolor{comment}{would print 12,345.}}
\DoxyCodeLine{84 \textcolor{comment}{}}
\DoxyCodeLine{85 \textcolor{comment}{For integers and floats, you can use a "{}\$"{} specifier and the number }}
\DoxyCodeLine{86 \textcolor{comment}{will be converted to float and then divided to get kilo, mega, giga or}}
\DoxyCodeLine{87 \textcolor{comment}{tera and then printed, so "{}\%\$d"{} 1024 is "{}1.0 k"{}, "{}\%\$.2d"{} 2536000 is }}
\DoxyCodeLine{88 \textcolor{comment}{"{}2.42 m"{}, etc.}}
\DoxyCodeLine{89 \textcolor{comment}{}}
\DoxyCodeLine{90 \textcolor{comment}{In addition to octal and hexadecimal conversions, you can print }}
\DoxyCodeLine{91 \textcolor{comment}{integers in binary: "{}\%b"{} for 256 would print 100.}}
\DoxyCodeLine{92 \textcolor{comment}{}}
